# Magnet Station

A cross-browser extension (Safari, Chrome, Firefox) for macOS and iOS that sends magnet links to Synology Download Station.

## Project Overview

This is a browser extension (Manifest V3) that allows users to click magnet links on any webpage and automatically add them to their Synology NAS Download Station. Safari is built via Xcode; Chrome and Firefox are built from a shared source directory via a build script.

## Tech Stack

- **Platform**: Browser Extension (Manifest V3) — Safari, Chrome, Firefox
- **Languages**: JavaScript, CSS, HTML, Swift (Safari native messaging only)
- **Build**: Xcode (Safari), `scripts/build.sh` (Chrome/Firefox)
- **Targets**: macOS, iOS (Safari); all platforms (Chrome/Firefox)
- **Tests**: Vitest (`npm test`)

## Key Features

1. **Login Methods**
   - Local IP/hostname with port
   - QuickConnect ID (resolves via Synology relay servers)

2. **Download Management**
   - View active downloads with progress
   - Delete downloads
   - Copy magnet links

3. **Quick Actions**
   - One-click magnet link adding (via content script)
   - Open Synology web UI in new tab
   - Refresh downloads list

4. **Cross-Browser Support**
   - Safari: iCloud Keychain for session/credentials via native messaging
   - Chrome/Firefox: storage.sync for cross-device session, browser password manager for credentials
   - Platform detection in `shared/platform.js`

5. **Buy Me a Coffee**
   - Donate banner in connected view header (Chrome/Firefox only in production)
   - Currently enabled on all platforms for debugging (TODO: restrict to non-Safari)

## Synology API

Uses DSM 7.x Web API:

- **Auth**: `/webapi/entry.cgi?api=SYNO.API.Auth&version=7`
- **Tasks**: `/webapi/DownloadStation/task.cgi?api=SYNO.DownloadStation.Task&version=1`

### QuickConnect Resolution

1. POST to `https://global.quickconnect.to/Serv.php` (follow regional redirects if errno 4)
2. Get server info (LAN IPs, DDNS, external IP, relay)
3. Try candidates in order: LAN (http then https) → DDNS → external IP → relay tunnel
4. On connection failure, re-resolve QuickConnect and try again

## Design System

CSS Variables (defined in popup.css):
- `--accent`: #D63031 (red - primary actions, delete, disconnect)
- `--blue`: #007AFF (secondary actions, refresh, copy, progress)
- `--success`: #00b894 (complete, seeding)
- `--warning`: #fdcb6e (paused)
- `--radius`: 10px
- `--transition`: 0.2s ease

Supports light and dark mode via `prefers-color-scheme`.

## Common Tasks

### Adding a new feature to popup
1. Edit files in `shared/` (popup.html, popup.css, popup.js)
2. Run `./scripts/build.sh all` to sync to all browser targets
3. Safari: rebuild in Xcode; Chrome/Firefox: reload in browser

### Building
```bash
./scripts/build.sh chrome   # → dist/chrome/
./scripts/build.sh firefox  # → dist/firefox/
./scripts/build.sh safari   # → Shared (Extension)/Resources/
./scripts/build.sh all      # all three
```

The build script inlines `platform.js` into popup.js and background.js (strips `export`/`import` keywords) since Safari doesn't support ES module imports in extension scripts.

### Testing extensions
- **Safari**: Build in Xcode (⌘R), enable in Safari → Settings → Extensions
- **Chrome**: `chrome://extensions` → Developer mode → Load unpacked → `dist/chrome/`
- **Firefox**: `about:debugging` → Load Temporary Add-on → `dist/firefox/manifest.json`
- **Unit tests**: `npm test` (Vitest)
- **Console**: Safari: Develop → Web Extension Background Page

### Updating icons
1. Replace source PNG/SVG
2. Generate sizes: 16, 32, 48, 64, 96, 128, 256, 512, 1024
3. Update `Contents.json` in asset catalogs
4. Clean build in Xcode (Cmd+Shift+K)

## Error Handling

- Session expiry (code 105) → auto-logout
- 2FA (code 403) → not supported, show message
- Connection timeout → 10 second limit
- User-friendly error messages in popup
